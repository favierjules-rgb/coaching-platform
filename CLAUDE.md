# Instructions permanentes du projet

## Skills UI/UX obligatoires

Pour tout chantier concernant le design, l’UI, l’UX, les animations, l’accessibilité ou le responsive :

1. Inspecter les skills locales présentes dans `.agents/skills/`.
2. Lire les fichiers `SKILL.md` pertinents avant de proposer ou modifier du code.
3. Utiliser notamment, lorsqu’elles sont adaptées :
   - animation-vocabulary
   - apple-design
   - emil-design-eng
   - find-animation-opportunities
   - improve-animations
   - review-animations
4. Indiquer brièvement quelles recommandations concrètes ont été appliquées.
5. Ne pas sur-animer.
6. Respecter `prefers-reduced-motion`.
7. Ne jamais modifier la logique métier pendant un chantier purement visuel.
8. Ne pas ajouter de nouvelle dépendance UI ou animation sans autorisation.
